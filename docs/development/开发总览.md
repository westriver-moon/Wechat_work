# 开发总览

## 文档定位

本文档面向当前仓库的维护者和二次开发者，说明项目的整体开发边界、目录职责、本地开发方式和协作入口。阅读顺序建议如下：

1. 先阅读本文件，建立整体认识。
2. 再按需要阅读后端、前端、AI、工单专项文档和接口参考文档。
3. 涉及部署和联调时，再阅读 `docs/deployment/` 与 `docs/testing/`。

## 项目目标

本项目服务于“北京理工大学新生助手”这一公众号 / Web 场景，当前实际落地的能力包括：

- 地点查询：通过腾讯地图完成地点搜索、坐标定位和地图预览。
- 智能问答：优先使用公开网页抓取证据，其次回退本地知识库，最终生成对新生问题的回答。
- 人工工单：为新生提供提问、学长答复、微信查询码回查、网页查看进度的完整闭环。
- 登录与权限：所有主要页面和 API 统一受 BIT 登录与本地权限模型控制。

## 当前技术栈

### 后端

- Python 3.10+
- Flask
- requests
- python-dotenv
- SQLite
- 可选 Gunicorn
- 可选 sentence-transformers + faiss-cpu

### 前端

- 原生 HTML / CSS / JavaScript
- 后端静态托管页面
- 腾讯地图 JavaScript API / WebService API

### 外部依赖

- BIT 统一身份认证（通过 `bit_login`）
- 腾讯位置服务
- DeepSeek 或 OpenAI 兼容接口
- 公开网页抓取来源

## 当前目录结构

项目开发时最需要关注的目录如下：

```text
app/
├── backend/
│   ├── app.py
│   ├── modules/
│   │   ├── auth/
│   │   ├── ai/
│   │   ├── feature3/
│   │   ├── pages/
│   │   └── shared/
│   └── tests/
├── data/
└── web/
```

目录职责：

- `app/backend/app.py`：Flask 应用唯一启动入口。
- `app/backend/modules/`：后端真实实现代码。
- `app/backend/tests/`：自动化冒烟测试脚本。
- `app/data/`：AI 知识库原始 Markdown 文档。
- `app/web/`：所有页面模板。
- `docs/history/`：历史开发、部署、测试记录归档区，只保留参考价值，不作为当前实施依据。

## 后端模块分工

### `modules/auth`

负责：

- BIT 登录接入
- Session 用户对象构造
- 权限位定义与判断
- 认证相关路由
- 登录拦截

### `modules/ai`

负责：

- AI 问答主链路
- LLM 调用
- 本地 FAQ / 知识库检索
- 全网搜索与公开网页抓取
- AI 相关接口

### `modules/feature3`

负责：

- 工单创建、查询、关闭
- 学长端答复
- worker 异步任务处理
- 微信回调与查询码逻辑
- 微信客服消息推送（可选）

### `modules/pages`

负责：

- 页面入口路由
- 静态 HTML 的访问控制

### `modules/shared`

负责：

- SQLite 数据访问
- backend 根目录 / data / web 等共享路径

## 本地开发建议流程

### 1. 进入后端目录

```powershell
Set-Location .\app\backend
```

### 2. 准备解释器

优先使用已验证可用的 Conda 环境：

```powershell
conda activate D:\envs\wechat_work
```

如果不用 Conda，也可以使用虚拟环境：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

### 3. 配置 `.env`

从 `.env.example` 复制得到 `.env`，按本机环境填写：

- `SECRET_KEY`
- `WECHAT_TOKEN`
- `DEEPSEEK_API_KEY` / `OPENAI_API_KEY`
- `DB_PATH`
- `FEATURE3_*`
- 腾讯地图 Key 需要单独写入 `app/web/place.html`

### 4. 启动服务

```powershell
D:\envs\wechat_work\python.exe .\app.py
```

### 5. 执行自动化验证

```powershell
D:\envs\wechat_work\python.exe .\tests\feature3_smoke_test.py
D:\envs\wechat_work\python.exe .\tests\url_fetch_smoke_test.py https://www.bit.edu.cn --search-query "北京理工大学 选课"
```

## 开发协作建议

### 修改后端时

- 优先定位到 `modules/` 下的真实实现文件。
- 不要在 backend 根目录新增兼容转发层。
- 新增功能时，优先考虑是否应归入现有模块，而不是随意增设平级入口文件。

### 修改前端时

- 确认对应页面由哪个路由暴露。
- 确认该页面是否受登录保护或权限保护。
- 提交前至少做一次页面打开和一次接口交互验证。

### 修改 AI 问答时

- 区分“网页抓取路径”和“知识库回退路径”。
- 不要在未评估的情况下扩大抓取范围到本机、内网或受限地址。
- 修改检索逻辑后务必重新验证 `/api/chat` 和 `/api/rebuild_index`。

### 修改工单系统时

- 注意问题状态流转：`pending -> answered -> closed`。
- 注意工单逻辑和登录权限逻辑是并存关系，不要互相覆盖。
- 微信文本逻辑和网页工单逻辑共享同一套持久化层。
