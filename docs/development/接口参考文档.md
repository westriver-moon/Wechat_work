# 接口参考文档

本文档面向当前项目的前后端联调、接口排查和二次开发，汇总当前页面入口、API 路由、请求参数、典型返回结构与权限要求。

## 1. 全局访问规则

### 1.1 公开路径

以下路径不经过登录拦截：

- `/login`
- `/logout`
- `/api/auth/*`
- `/wechat`
- `/favicon*`

### 1.2 默认登录要求

除上述公开路径外，其余页面和 API 默认都需要已登录会话。

未登录时：

- 页面请求：302 跳转到 `/login?next=...`
- API 请求：返回 `401`

典型返回：

```json
{
  "error": "authentication required",
  "login_url": "/login"
}
```

### 1.3 权限位

当前后端使用整数位权限：

- `PERM_USE_APP = 1`：基础使用权限
- `PERM_ANSWER = 2`：学长答复权限
- `PERM_ADMIN = 4`：管理员权限

说明：

- 普通登录用户默认拥有 `PERM_USE_APP`
- `senior` 页面和工单处理接口需要 `PERM_ANSWER`
- `feature3_status` 会把当前用户权限状态放入返回载荷

### 1.4 当前登录用户对象

后端会在 session 中保存 `auth_user`，其典型结构为：

```json
{
  "student_code": "2025xxxx",
  "name": "张三",
  "major": "计算机科学与技术",
  "college": "计算机学院",
  "grade": "2025",
  "class_name": "01班",
  "permissions": 1,
  "permission_labels": ["use_app"],
  "owner_identity": "user:...",
  "is_privileged": false
}
```

## 2. 页面路由

### `GET /`

用途：站点首页入口。

权限：需要登录。

行为：

- 未登录时跳转到 `/login?next=/`
- 已登录时返回 `index.html`

### `GET /login`

用途：登录页。

权限：公开。

行为：

- 未登录时返回 `login.html`
- 已登录时跳转到 `next` 参数或 `/`

### `GET /logout`

用途：退出登录。

权限：公开。

行为：清理当前 session 并跳转到 `/login`

### `GET /chat`

用途：AI 问答页。

权限：需要登录。

返回：`chat.html`

### `GET /place`

用途：地点查询页。

权限：需要登录。

返回：`place.html`

### `GET /freshman`

用途：新生提问页。

权限：需要登录。

返回：`freshman.html`

### `GET /senior`

用途：学长答复台。

权限：需要 `PERM_ANSWER`。

行为：

- 有权限时返回 `senior.html`
- 无权限时页面请求会被重定向到 `/`

## 3. 认证接口

### `POST /api/auth/login`

用途：执行 BIT 登录并写入当前 session。

权限：公开。

请求体：

```json
{
  "username": "学号或账号",
  "password": "密码",
  "next": "/chat"
}
```

字段说明：

- `username`：必填
- `password`：必填
- `next`：可选，登录成功后前端可据此跳转

成功返回：`200`

```json
{
  "ok": true,
  "user": {
    "student_code": "2025xxxx",
    "name": "张三",
    "permissions": 1,
    "permission_labels": ["use_app"],
    "owner_identity": "user:...",
    "is_privileged": false
  },
  "next": "/chat"
}
```

失败返回：`401`

```json
{
  "ok": false,
  "error": "登录失败: 账号或密码错误"
}
```

### `POST /api/auth/logout`

用途：退出当前登录状态。

权限：公开。

请求体：无。

成功返回：`200`

```json
{
  "ok": true
}
```

### `GET /api/auth/me`

用途：读取当前登录用户。

权限：需要登录。

成功返回：`200`

```json
{
  "user": {
    "student_code": "2025xxxx",
    "name": "张三",
    "permissions": 1,
    "permission_labels": ["use_app"],
    "owner_identity": "user:...",
    "is_privileged": false
  }
}
```

未登录返回：`401`

```json
{
  "error": "authentication required"
}
```

## 4. AI 接口

### `POST /api/chat`

用途：AI 主问答接口。

权限：需要登录。

请求体：

```json
{
  "question": "北京理工大学新生报到需要带哪些材料？"
}
```

字段说明：

- `question`：必填，用户问题文本

成功返回：`200`

```json
{
  "answer": "...",
  "steps": [
    {
      "stage": "search",
      "ok": true,
      "message": "已启动全网搜索：北京理工大学 选课",
      "url": ""
    }
  ],
  "web_sources": [
    {
      "url": "https://...",
      "final_url": "https://...",
      "title": "页面标题",
      "excerpt": "摘要",
      "ok": true
    }
  ],
  "mode": "web"
}
```

返回字段说明：

- `answer`：最终回答文本，当前为 Markdown 风格字符串
- `steps`：处理过程日志，便于前端展示过程或调试
- `web_sources`：本次成功抓取到的网页结果列表
- `mode`：`web` 或 `kb`

失败返回：

- 缺少问题：`400`

```json
{
  "error": "question is required"
}
```

### `POST /api/rebuild_index`

用途：重建本地知识库索引。

权限：需要登录。

请求体：无。

成功返回：`200`

```json
{
  "ok": true,
  "ready": true,
  "backend": "lexical",
  "doc_count": 103,
  "error": ""
}
```

失败返回：`500`

```json
{
  "ok": false,
  "error": "具体异常信息"
}
```

### `GET /api/kb_query?q=...`

用途：直接查看本地知识库检索结果与可选 LLM 回答。

权限：需要登录。

查询参数：

- `q`：必填，问题文本

成功返回：`200`

```json
{
  "query": "新生什么时候选课？",
  "retrieved": "(来源:faq.md score:0.428) ...",
  "top_score": 0.428,
  "answer": "..."
}
```

说明：

- `retrieved` 是后端拼接后的检索参考文本，不是结构化数组
- 若当前未启用 LLM，`answer` 可能为空字符串

缺少参数返回：`400`

```json
{
  "error": "q is required"
}
```

## 5. 工单接口

### `POST /api/questions`

用途：新生创建问题。

权限：需要登录。

请求体：

```json
{
  "title": "宿舍问题",
  "content": "宿舍什么时候可以入住？"
}
```

字段说明：

- `title`：可选
- `content`：必填

成功返回：`201`

```json
{
  "id": 12,
  "status": "pending",
  "view_code": "AB12CD34",
  "message": "已收到，问题编号：12。请在“我的问题”页面查看进展。"
}
```

失败返回：

- 缺少内容：`400`
- 会话异常：`401`

### `PUT /api/questions/<question_id>/close`

用途：关闭自己的问题。

权限：需要登录。

请求体：

```json
{
  "view_code": "AB12CD34"
}
```

说明：

- 后端支持两种关闭凭据：当前会话中的 `owner_identity`，或显式传入 `view_code`
- 页面端通常直接带当前登录态关闭；匿名追踪场景可使用 `view_code`

成功返回：`200`

```json
{
  "status": "closed"
}
```

失败返回：

- 未提供 `owner_identity` 和 `view_code`：`400`
- 校验失败：`403`

## 6. 微信接口

### `GET /wechat`

用途：微信公众号服务器校验。

权限：公开。

查询参数：

- `signature`
- `timestamp`
- `nonce`
- `echostr`

成功返回：

- 签名通过：原样返回 `echostr`
- 签名失败：`403 forbidden`

### `POST /wechat`

用途：接收微信公众号文本消息。

权限：公开。

请求体：微信公众号 XML。

处理逻辑：

- 先校验签名
- 只处理 `MsgType=text`
- 非文本消息直接返回 `success`
- 文本消息会走 `build_wechat_reply_text` 生成回复，再返回 XML

成功返回：

- 文本消息：`application/xml`
- 其他消息或异常：`success`

## 7. 联调建议

- 前端联调时，优先先调 `GET /api/auth/me` 确认会话是否存在。
- AI 接口联调时，建议同时观察 `mode`、`steps` 和 `web_sources`，不要只看 `answer`。
- 学长端联调时，先确认当前登录用户是否具备 `PERM_ANSWER`，否则 `senior` 页面和相关 API 都会受限。
- 微信联调时，先完成公网可达、Token 一致和 Nginx/HTTPS 配置，再测消息回调。
