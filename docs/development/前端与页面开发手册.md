# 前端与页面开发手册

## 1. 页面目录与托管方式

前端页面位于 `app/web/`，由 Flask 通过静态目录直接提供。

当前页面：

- `index.html`：统一入口页
- `login.html`：登录页
- `chat.html`：AI 问答页
- `place.html`：地点查询页
- `freshman.html`：新生工单页
- `senior.html`：学长处理台

页面本身不经过模板渲染，后端只负责把对应 HTML 文件送出。因此前端开发时需要注意：

- 页面状态主要靠浏览器端脚本维护
- 认证状态依赖后端 Session
- 页面内数据基本来自 API

## 2. 页面访问控制

### 公开路径

当前公开路径主要包括：

- `/login`
- `/logout`
- `/api/auth/*`
- `/wechat`
- favicon

### 受保护页面

以下页面默认要求登录：

- `/`
- `/chat`
- `/place`
- `/freshman`
- `/senior`

其中 `/senior` 还需要答复权限。

## 3. 页面职责

### `login.html`

负责：

- 展示 BIT 登录入口
- 提交 `POST /api/auth/login`
- 登录成功后跳转到 `next` 或首页

开发注意：

- 不要在页面里处理真实权限逻辑
- 错误提示来自接口返回
- 如果改登录流程，必须同步验证 `/api/auth/me`

### `index.html`

负责：

- 提供统一入口导航
- 让用户进入 AI、地图、工单等页面
- 展示系统的首页信息

### `chat.html`

负责：

- 提交问题到 `POST /api/chat`
- 渲染 AI 最终回答
- 提供快捷提问或状态提示

开发注意：

- 当前页面应聚焦“提问 -> 回答”主流程
- 不建议重新引入复杂的旧流程可视化面板
- 回答来源和抓取过程展示应保持克制，避免压过核心内容

### `place.html`

负责：

- 使用腾讯地图进行地点查询
- 展示地点名称、地址、坐标和地图预览

开发注意：

- 需要真实腾讯地图 Key
- Key 未替换时页面可以打开，但真实地点搜索会失败
- 若新增地图交互，优先保持移动端可用性

### `freshman.html`

负责：

- 新生提交问题
- 查看“我的问题”列表
- 用问题编号 + 查询码查询
- 关闭已答复工单

开发注意：

- 该页面的状态要和后端工单状态一致
- 提交成功后应明确展示问题编号和查询码
- 自动轮询存在时，要避免引起页面抖动和重复提醒

### `senior.html`

负责：

- 展示待处理问题
- 提交答复
- 查看已答复问题
- 做搜索和筛选

开发注意：

- 该页面必须受权限保护
- 提交答复后要及时刷新待办和已答复列表
- 如果后续扩展批量处理或工单标签，优先从这里扩展

## 4. 前端与后端 API 对应关系

### 登录与身份

- `POST /api/auth/login`
- `POST /api/auth/logout`
- `GET /api/auth/me`

### AI

- `POST /api/chat`
- `POST /api/rebuild_index`
- `GET /api/kb_query`
- `GET /api/demo_status`

### 工单

- `POST /api/questions`
- `GET /api/questions/mine`
- `GET /api/questions/track`
- `PUT /api/questions/<id>/close`
- `GET /api/tasks/pending`
- `GET /api/tasks/answered`
- `POST /api/answers`
- `GET /api/feature3_status`

## 5. 前端开发原则

### 保持页面职责单一

- 登录页只处理登录
- 首页只处理导航与介绍
- AI 页只处理问答
- 地图页只处理地点查询
- 工单页只处理问题生命周期

### 减少对实现细节的假设

前端不要依赖：

- 数据库内部字段命名
- worker 是否开启
- AI 当前具体用的是网页抓取还是知识库回退

前端只应依赖 API 合同和必要的状态字段。

### 移动端优先

该项目主要场景仍然偏向公众号内 H5，因此：

- 交互布局需要优先考虑小屏
- 长文本回答要考虑滚动和换行
- 表单和按钮要保证触控友好

## 6. 常见前端联调问题

### 页面能打开但接口 401

说明当前页面受登录保护，但浏览器没有有效会话。优先检查：

- 是否已完成登录
- Session 是否被清掉
- 是否直接访问了受保护接口

### 地图页报 Key 错误

优先检查：

- `app/web/place.html` 中是否仍为占位符
- 腾讯控制台是否已开通对应服务
- 域名限制或配额是否触发

### AI 页无回答或超时

优先检查：

- `POST /api/chat` 返回码
- `/api/demo_status` 中 `llm_enabled` 与 `kb_index_ready`
- 浏览器是否因为超时中断请求

### 工单页列表异常

优先检查：

- `/api/questions/mine` 的返回
- 当前登录用户的 `owner_identity`
- 问题是否实际写入数据库

## 7. 前端改动后的最低验证要求

每次修改页面后，至少执行以下验证：

1. 页面能正常打开
2. 页面使用的 API 至少成功调用一次
3. 页面在移动端宽度下无明显错位
4. 登录保护没有被破坏

如果改动涉及 AI 或工单主链路，还应追加一次对应的冒烟或手工回归。
