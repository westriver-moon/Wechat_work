# 部署总览

## 文档定位

本文档用于说明当前项目支持的运行形态、部署拓扑、依赖项和部署文档入口。它不是具体操作手册，而是部署前先读的总说明。

## 1. 支持的运行形态

当前项目主要支持两类运行形态：

### 1. 本地开发 / 联调

适用场景：

- 开发阶段调试
- 浏览器回环验证
- API 调试
- 冒烟测试执行

典型方式：

- Windows 本地运行
- 使用 Conda 或 `.venv`
- 通过 `python app.py` 启动
- 访问 `http://127.0.0.1:5000`

### 2. Linux 生产部署

适用场景：

- 腾讯云 / Ubuntu 服务器上线
- 公网域名访问
- 微信公众号回调接入
- Nginx + Gunicorn + systemd 托管

## 2. 当前推荐部署拓扑

```text
用户浏览器 / 微信公众号
        |
        v
      Nginx
        |
        v
  Gunicorn / Flask
        |
        v
     SQLite / 本地运行文件
```

说明：

- Nginx 处理 80 / 443、HTTPS 和反向代理
- Gunicorn 托管 Flask 应用
- Flask 应用读取 `app/backend/.env`
- SQLite 存储登录附加身份、工单、任务等状态
- AI 模块还会读写知识库索引状态文件

## 3. 部署前必须准备的外部条件

### 基础条件

- 可运行 Python 3.10+ 的环境
- 可写的 `app/backend/` 目录
- 正确配置的 `.env`

### 外部服务 / 密钥

- BIT 登录依赖：`bit_login` 可用安装来源
- 公众号 Token：`WECHAT_TOKEN`
- 腾讯地图 Key：写入 `app/web/place.html`
- LLM Key：`DEEPSEEK_API_KEY` 或 `OPENAI_API_KEY`（可选）
- 微信主动推送：`WECHAT_APPID`、`WECHAT_APPSECRET`（可选）

### 生产环境额外要求

- 域名
- HTTPS 证书
- 放通 80 / 443
- 微信公众号后台可配置服务器 URL

## 4. backend 根目录中与部署相关的关键文件

- `app.py`：应用入口
- `.env.example`：环境变量模板
- `requirements.txt`：基础依赖
- `requirements-rag.txt`：RAG 相关附加依赖
- `requirements-prod.txt`：生产附加依赖
- `auth_identities.json.example`：特权身份模板
- `official_url_allowlist.json`：保留的允许站点配置文件

## 5. 运行期文件说明

生产或本地运行后，backend 目录中常见运行产物：

- `.env`
- `feature3.db`
- `kb_meta.json`
- `kb_state.json`
- `kb_texts.json`
- `wechat_token_cache.json`

这些文件必须保证进程有读写权限。

## 7. 部署原则

- backend 根目录只保留主入口和运行配置，不再人为增加兼容层或临时脚本
- 生产部署优先保证登录、AI、工单三条主链路同时可用
- 优先用自动化验证和最小手工验收闭环，而不是只看进程是否启动
- 当外部依赖不可用时，要区分“代码错误”和“外部配置缺失”

## 8. 推荐部署顺序

### 本地验证顺序

1. 配置 `.env`
2. 启动 `app.py`
3. 跑两条冒烟脚本
4. 浏览器验证登录、AI、工单页面

### 生产验证顺序

1. 完成 Python 环境与依赖安装
2. 配置 `.env`
3. 本机验证 Gunicorn
4. 配置 systemd
5. 配置 Nginx 与 HTTPS
6. 配置公众号后台 URL 与菜单
7. 执行验收测试
