# 测试与验收手册

## 1. 文档目标

本文档用于统一当前项目的自动化测试、接口验证、页面回归和部署验收方式。它替代旧的分散测试说明，作为当前唯一推荐测试入口。

## 2. 测试范围

当前测试范围覆盖以下模块：

- 登录与权限拦截
- AI 问答主链路
- 网页抓取与全网搜索
- 人工工单主流程
- 微信回调基础链路
- 页面回环访问

## 3. 自动化测试

当前仓库已有两条自动化冒烟测试：

### 工单主流程冒烟测试

文件：`app/backend/tests/feature3_smoke_test.py`

覆盖内容：

1. 创建问题
2. 查询“我的问题”
3. 学长提交答复
4. 编号 + 查询码回查
5. 已答复列表搜索
6. 关闭问题
7. 关闭后仍出现在已答复列表
8. 关闭后拒绝再次答复

执行方式：

```powershell
Set-Location .\app\backend
D:\envs\wechat_work\python.exe .\tests\feature3_smoke_test.py
```

### 网页抓取与全网搜索冒烟测试

文件：`app/backend/tests/url_fetch_smoke_test.py`

覆盖内容：

- 指定公开 URL 抓取
- 全网搜索候选抓取
- 候选页实际抓取
- 抓取结果摘要输出

执行方式：

```powershell
Set-Location .\app\backend
D:\envs\wechat_work\python.exe .\tests\url_fetch_smoke_test.py https://www.bit.edu.cn --search-query "北京理工大学 选课"
```

## 4. 启动前检查

执行任何测试前，至少确认：

- 当前目录为 `app/backend`
- `.env` 已存在
- 所需依赖已安装
- 腾讯地图 Key 已按需要替换
- 启动解释器正确

推荐启动命令：

```powershell
D:\envs\wechat_work\python.exe .\app.py
```

## 5. 本地浏览器回环测试

### 登录与入口

1. 打开 `http://127.0.0.1:5000/`
2. 确认未登录时跳转 `/login`
3. 登录页能正常渲染
4. 登录成功后能进入首页或 `next` 指向页面

### AI 页面

1. 进入 `/chat`
2. 提交一个普通问题
3. 确认页面显示回答
4. 如果开启网页抓取，确认至少一次返回可用来源

### 地图页面

1. 进入 `/place`
2. 确认页面能打开
3. 若已配置真实地图 Key，则再验证真实地点查询

### 工单页面

1. 进入 `/freshman`
2. 提交一条问题
3. 刷新列表，确认问题存在
4. 具备答复权限的账号进入 `/senior`
5. 提交答复
6. 回到 `/freshman` 确认问题状态变化

## 8. 问题记录建议

每次测试建议至少记录：

- 测试时间
- 测试环境（本地 / 服务器）
- 使用的分支或代码版本
- 失败现象
- 是否可稳定复现
- 初步归因（代码 / 外部依赖 / 配置 / 数据）

## 9. 常见判定标准

### 属于代码问题

- 同一环境稳定复现
- 不依赖外部密钥缺失也会失败
- 自动化测试直接失败
- 日志中出现明确异常堆栈

### 属于外部配置问题

- 腾讯地图 Key 缺失或无效
- GitHub 源导致登录依赖安装失败
- LLM Key 缺失或额度问题
- 微信后台 Token / URL 不一致

### 属于环境问题

- 当前目录错误
- 5000 端口已有旧监听
- Python 环境不对
- 依赖版本不兼容
