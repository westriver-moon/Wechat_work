# 服务器部署环境清单与步骤（功能一 + 功能二，含网页演示模式，已支持 HuggingFace 向量检索）

适用范围：当前仓库中已实现的两个功能
- 功能一：地点查询（腾讯地图）
- 功能二：智能问答（FAQ + 可选 DeepSeek）

不包含：功能三（人工服务工单系统）

---

## 1. 目标架构

- 用户访问 `https://你的域名/`
- Nginx 监听 80/443
- Nginx 反向代理到本机 `127.0.0.1:5000`
- Python 后端（Flask + Gunicorn）提供：
  - `/` 统一入口
  - `/place` 地点查询页面
  - `/chat` 智能问答页面
  - `/api/chat` 问答接口
  - `/api/kb_query` 检索调试接口
  - `/api/rebuild_index` 索引重建接口
  - `/api/demo_status` 演示环境自检接口
  - `/wechat` 公众号服务器接口

---

## 2. 环境清单

### 2.1 服务器基础

- Ubuntu 24.04 LTS
- 公网 IP
- 已备案域名（用于微信和 HTTPS）
- 可执行 sudo 的系统账户

### 2.2 必装软件

- Nginx（APT 安装）
- Python 3.10+（Ubuntu 24.04 默认 Python 3.12）
- pip（随 Python）
- systemd（系统默认，用于托管 Gunicorn）

### 2.3 网络与防火墙

- 放行入站端口：`80`、`443`
- 本机回环端口：`127.0.0.1:5000`（仅本机访问）
- 域名 A 记录指向服务器公网 IP

### 2.4 Python 依赖

在 `app/backend/requirements.txt`：
- Flask
- python-dotenv
- requests
- gunicorn
- sentence-transformers
- faiss-cpu
- numpy

## 3. 公众号与第三方账号准备

### 3.1 微信公众号后台

- 开启开发者模式
- 配置服务器地址：`https://你的域名/wechat`
- 设置 Token（需与后端 `.env` 一致）
- 菜单入口可配：
  - `https://你的域名/`
  - 或分别配置 `/place`、`/chat`

### 3.2 腾讯地图（功能一）

- 申请腾讯位置服务 Key
- 开通服务：JavaScript API + WebService API + StaticMap
- 为 Key 配置可用域名/来源限制
- 关注每日配额（超限会返回状态码 121）

### 3.3 DeepSeek（功能二）

- 准备 API Key
- 推荐地址：`https://api.deepseek.com`
- 兼容地址：`https://api.deepseek.com/v1`（可用，但与模型版本无关）
- 建议确保账号有可用额度

## 4. 项目部署步骤（Ubuntu 24.04 LTS）

### 步骤 1：上传代码

将仓库上传到服务器项目根目录，例如：
- `/opt/wechat_work`

### 步骤 2：安装 Python 依赖

执行：

```bash
cd /opt/wechat_work
cd app/backend
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 步骤 3：配置后端环境变量

在 `app/backend/.env` 填写：

```env
FLASK_ENV=development
PORT=5000
WECHAT_TOKEN=你的公众号token
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_API_KEY=你的deepseek_key
DEEPSEEK_MODEL=deepseek-chat

# OpenAI 兼容备用（可留空）
OPENAI_BASE_URL=
OPENAI_API_KEY=
OPENAI_MODEL=

OPENAI_TIMEOUT=15
LLM_TEMPERATURE=0.2
LLM_TOP_P=0.9
LLM_MAX_TOKENS=400
LLM_MIN_CONFIDENCE=0.60

# 推荐值：0（优先 HuggingFace + FAISS）
# 备用值：1（网络异常时回退词法检索）
KB_FORCE_LEXICAL=0
```

说明：
- 当前后端优先读取 `DEEPSEEK_*` 变量。
- 若不使用大模型，留空 `DEEPSEEK_API_KEY` 与 `OPENAI_API_KEY`，系统将仅使用 FAQ/检索兜底回复。
- `WECHAT_TOKEN` 必须和公众号后台一致。

### 步骤 4：启动并验证后端

```bash
cd /opt/wechat_work
cd app/backend
source .venv/bin/activate
gunicorn -w 2 -b 127.0.0.1:5000 app:app
```

浏览器访问本机：
- `http://127.0.0.1:5000/`
- `http://127.0.0.1:5000/chat`
- `http://127.0.0.1:5000/place`

接口自检：

```bash
curl 'http://127.0.0.1:5000/api/demo_status'
curl -X POST 'http://127.0.0.1:5000/api/rebuild_index'
```

预期：
- `/api/rebuild_index` 返回 `ok=true`
- `/api/demo_status` 返回 provider、llm_enabled、model、kb_index_ready、kb_backend、kb_doc_count 字段。
- 当 `KB_FORCE_LEXICAL=0` 且网络可用时，`kb_backend` 期望为 `faiss`。
- 首次重建可能耗时 30~90 秒（模型加载/缓存），请耐心等待。

### 步骤 5：配置 Nginx

- 复制 `deploy/nginx/wechat_assistant.conf` 到：
  - `/etc/nginx/sites-available/wechat_assistant.conf`
- 软链接到：
  - `/etc/nginx/sites-enabled/wechat_assistant.conf`
- 修改：
  - `server_name` 为你的域名
  - 证书路径 `ssl_certificate`、`ssl_certificate_key`

验证并重载：

```bash
sudo nginx -t
sudo systemctl restart nginx
```

首次启动（如果尚未启动）：

```bash
sudo systemctl start nginx
```

### 步骤 6：配置 HTTPS 证书

- 准备域名证书文件（PEM）
- 在 Nginx 配置中启用 443
- 确保 80 自动跳转到 443

### 步骤 7：注册后端为系统服务（推荐）

按仓库文档 `deploy/linux/systemd-service.md`：
- 创建 `wechat-assistant.service`
- 执行 `sudo systemctl enable --now wechat-assistant`
- 重启服务器后自动拉起

## 6. 常见故障定位

### 6.1 地图 API 失败

- 错误码 `121`：Key 当日配额用尽
- 错误码 `311`：Key 无效或服务未开通
- 处理建议：
  - 到腾讯位置服务控制台检查服务开通与配额
  - 检查域名白名单是否包含当前访问域名

### 6.2 DeepSeek 无响应

- `.env` 未正确加载（后端固定读取 backend 目录下 `.env`）
- `DEEPSEEK_API_KEY` 无效或额度不足
- 网络波动导致超时（可提高 `OPENAI_TIMEOUT`）
- 先调用 `/api/demo_status` 确认 provider 与 llm_enabled 状态
- 查看后端日志中的 `LLM request failed/exception` 信息

### 6.3 RAG 索引未生效

- `/api/demo_status` 显示 `kb_index_ready=false`：先调用 `/api/rebuild_index`
- `kb_doc_count=0`：检查 `app/data` 是否有可索引的 `.md` 文档
- 若服务器外网不稳定导致向量模型下载超时：启用 `KB_FORCE_LEXICAL=1` 保障检索可用
- 若 `/api/rebuild_index` 首次调用较慢：优先判断是否仍在构建中，不要重复并发触发重建

### 6.4 公众号验证失败

- `WECHAT_TOKEN` 不一致
- URL 不是公网 HTTPS
- Nginx 未转发 `/wechat` 到后端
