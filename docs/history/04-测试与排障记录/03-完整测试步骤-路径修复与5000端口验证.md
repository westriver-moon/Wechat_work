# 路径报错原因与完整测试步骤（Windows，本地5000端口）



## 2. 立即可用的修复命令

### 2.1 在 CMD里执行

cd /d <项目根目录>
cd app\backend

说明：
- `/d` 可同时切换盘符和目录，最稳妥。

Set-Location <项目根目录>
Set-Location .\app\backend

## 3. 完整测试步骤（从启动到验证）

以下流程默认你使用 PowerShell（推荐），并在本机 `5000` 端口验证。

### 步骤 0：确认目录存在

Test-Path .\app\backend

预期：返回 `True`。

### 步骤 1：进入后端目录

Set-Location <项目根目录>
Set-Location .\app\backend
Get-Location

预期：输出路径是
`<项目根目录>\app\backend`

### 步骤 2：安装依赖（首次或依赖更新后）

.\.venv\Scripts\python.exe -m pip install -r .\requirements.txt

### 步骤 3：检查 .env 关键配置

至少确认以下项：

- `PORT=5000`
- `DEEPSEEK_BASE_URL=https://api.deepseek.com`
- `DEEPSEEK_API_KEY=你的有效key`
- `DEEPSEEK_MODEL=deepseek-chat`
- `KB_FORCE_LEXICAL=0`

### 步骤 4：启动后端

.\.venv\Scripts\python.exe .\app.py

保持该终端运行，不要关闭。

### 步骤 5：新开一个终端做接口测试

先进入同一目录：

Set-Location <项目根目录>
Set-Location .\app\backend

#### 5.1 状态接口

Invoke-RestMethod -Method Get -Uri 'http://127.0.0.1:5000/api/demo_status'

预期关键字段：
- `kb_index_ready = true`
- `kb_backend = faiss`（当 `KB_FORCE_LEXICAL=0` 且网络可用）
- `kb_doc_count >= 1`

#### 5.2 重建索引

Invoke-RestMethod -Method Post -Uri 'http://127.0.0.1:5000/api/rebuild_index' -TimeoutSec 300

预期关键字段：
- `ok = true`
- `ready = true`
- `backend = faiss`
- `doc_count = 13`（当前仓库数据）

#### 5.3 聊天接口（中文，UTF-8 安全写法）

$json = '{"question":"新生军训一般有哪些安排？"}'
$bytes = [System.Text.Encoding]::UTF8.GetBytes($json)
Invoke-RestMethod -Method Post -Uri 'http://127.0.0.1:5000/api/chat' -ContentType 'application/json; charset=utf-8' -Body $bytes

预期：
- 返回 `answer` 字段。
- 回答应包含来源信息（例如 `08_军训与国防教育.md`）。

## 4. 常见问题排查

### 4.1 还是提示找不到路径

依次检查：

1. 是否先进入了项目根目录，再进入 `app\backend`。
2. `app\backend` 这一段是否拼写正确。
3. 是否在 CMD 里忘了加 `/d`。

### 4.2 /api/chat 中文变成问号

PowerShell 直接发中文 JSON 有时会乱码。使用上面的 UTF-8 字节体写法即可。

### 4.3 /api/rebuild_index 变慢

首次重建可能较慢（常见 30~90 秒）。后续通常会更快，这是正常现象。
