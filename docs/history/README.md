# 历史文档归档说明

本目录用于保存项目历史开发过程记录、阶段性设计稿、旧版部署说明和专项测试 / 排障文档。

## 归档原则

- 不删除已有历史记录
- 不把历史文档继续作为当前实施依据
- 当前有效文档统一位于 `docs/development/`、`docs/deployment/`、`docs/testing/`
- 历史文档按主题分组，并在同组内使用数字前缀保持阅读顺序

## 当前目录分组

- `01-功能方案与阶段说明/`：阶段性方案、MVP 说明
- `02-开发过程记录/`：旧版开发文档与专项开发过程记录
- `03-部署过程记录/`：旧版部署清单、Linux 上线说明
- `04-测试与排障记录/`：历史测试清单、故障定位与路径修复记录

## 命名规范

- 目录使用 `NN-主题名称`
- 文件使用 `NN-文档名称.md`
- 文件名前缀只表示当前归档目录中的阅读顺序，不代表严格时间线

## 使用建议

- 需要追溯设计演进、方案变化或历史排障过程时，优先查看本目录
- 需要按当前代码进行开发、部署、测试时，不要以本目录文档为主依据
- 如需当前接口、权限和页面说明，请直接查看 `docs/development/接口参考文档.md`
