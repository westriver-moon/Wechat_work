# 服务器部署环境清单与步骤（功能一 + 功能二）

适用范围：当前仓库中已实现的两个功能
- 功能一：地点查询（腾讯地图）
- 功能二：智能问答（FAQ + 可选 DeepSeek）

不包含：功能三（人工服务工单系统）

---

## 1. 目标架构

- 用户访问 `https://你的域名/`
- Nginx 监听 80/443
- Nginx 反向代理到本机 `127.0.0.1:5000`
- Python 后端（Flask + Waitress）提供：
  - `/` 统一入口
  - `/place` 地点查询页面
  - `/chat` 智能问答页面
  - `/api/chat` 问答接口
  - `/wechat` 公众号服务器接口

---

## 2. 环境清单

### 2.1 服务器基础

- Windows Server（建议 2019/2022）
- 公网 IP
- 已备案域名（用于微信和 HTTPS）
- 管理员权限账户

### 2.2 必装软件

- Nginx（建议放在 `C:\nginx`）
- Python 3.10+（可用 `py` 启动器）
- pip（随 Python）
- NSSM（用于把 Waitress 注册为服务）

### 2.3 网络与防火墙

- 放行入站端口：`80`、`443`
- 本机回环端口：`127.0.0.1:5000`（仅本机访问）
- 域名 A 记录指向服务器公网 IP

### 2.4 Python 依赖

在 `feature2-ai/backend/requirements.txt`：
- Flask
- python-dotenv
- requests
- waitress

---

## 3. 公众号与第三方账号准备

### 3.1 微信公众号后台

- 开启开发者模式
- 配置服务器地址：`https://你的域名/wechat`
- 设置 Token（需与后端 `.env` 一致）
- 菜单入口可配：
  - `https://你的域名/`
  - 或分别配置 `/place`、`/chat`

### 3.2 腾讯地图（功能一）

- 申请腾讯位置服务 Key
- 开通服务：JavaScript API + WebService API + StaticMap
- 为 Key 配置可用域名/来源限制
- 关注每日配额（超限会返回状态码 121）

### 3.3 DeepSeek（功能二，可选）

- 准备 API Key
- 使用 OpenAI 兼容地址：`https://api.deepseek.com/v1`
- 建议确保账号有可用额度

## 4. 项目部署步骤（Windows）

### 步骤 1：上传代码

将仓库上传到服务器目录，例如：
- `C:\apps\Wechar_Develop`

### 步骤 2：安装 Python 依赖

在 PowerShell 执行：

```powershell
cd C:\apps\Wechar_Develop\feature2-ai\backend
py -m pip install -r requirements.txt
```

### 步骤 3：配置后端环境变量

在 `feature2-ai/backend/.env` 填写：

```env
FLASK_ENV=development
PORT=5000
WECHAT_TOKEN=你的公众号token
OPENAI_BASE_URL=https://api.deepseek.com/v1
OPENAI_API_KEY=你的deepseek_key
OPENAI_MODEL=deepseek-chat
OPENAI_TIMEOUT=15
```

说明：
- 不使用 DeepSeek 时，可留空 `OPENAI_BASE_URL` 和 `OPENAI_API_KEY`，系统会仅用 FAQ 回复。
- `WECHAT_TOKEN` 必须和公众号后台一致。

### 步骤 4：启动并验证后端

```powershell
cd C:\apps\Wechar_Develop\feature2-ai\backend
py -m waitress --host=127.0.0.1 --port=5000 app:app
```

浏览器访问本机：
- `http://127.0.0.1:5000/`
- `http://127.0.0.1:5000/chat`
- `http://127.0.0.1:5000/place`

### 步骤 5：配置 Nginx

- 复制 `deploy/nginx/wechat_assistant.conf` 到：
  - `C:\nginx\conf\conf.d\wechat_assistant.conf`
- 修改：
  - `server_name` 为你的域名
  - 证书路径 `ssl_certificate`、`ssl_certificate_key`

验证并重载：

```powershell
cd C:\nginx
.\nginx.exe -t
.\nginx.exe -s reload
```

首次启动：

```powershell
.\nginx.exe
```

### 步骤 6：配置 HTTPS 证书

- 准备域名证书文件（PEM）
- 在 Nginx 配置中启用 443
- 确保 80 自动跳转到 443

### 步骤 7：注册后端为系统服务（推荐）

按仓库文档 `deploy/windows/nssm-service.md`：
- 使用 NSSM 注册 `WechatAssistant`
- 启动类型设为自动
- 重启服务器后自动拉起

## 6. 常见故障定位

### 6.1 地图 API 失败

- 错误码 `121`：Key 当日配额用尽
- 错误码 `311`：Key 无效或服务未开通
- 处理建议：
  - 到腾讯位置服务控制台检查服务开通与配额
  - 检查域名白名单是否包含当前访问域名

### 6.2 DeepSeek 无响应

- `.env` 未正确加载（已在代码中固定加载 backend 下 `.env`）
- API Key 无效或额度不足
- 网络波动导致超时（可提高 `OPENAI_TIMEOUT`）
- 查看后端日志中的 `LLM request failed/exception` 信息

### 6.3 公众号验证失败

- `WECHAT_TOKEN` 不一致
- URL 不是公网 HTTPS
- Nginx 未转发 `/wechat` 到后端
